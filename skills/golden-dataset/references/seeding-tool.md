# 投入ツールの生成（両フェーズ共通の機構）

ツールは両フェーズで共通の機構を持ち、フェーズ A では現行テスト環境、フェーズ B では新側環境をターゲットにする。

## ツールの位置づけ

- 投入ツールは**対象プロジェクト側に生成される成果物**であり、スキル本体に同梱する配布物ではない。したがって配布スキルの拡張子規約（`skills/**` は `.mjs`）の対象外で、**プロジェクト側の TypeScript（または SQL）として置く**
- スキルに同梱するのは「投入ツールを生成する手順・テンプレート」であって、ツール本体ではない
- 配置先は設定の `dataset_tool_dir`（未指定時は `seed/`）

## 言語の選択

- **TypeScript が既定**（本スキル群の前提スタック。新側が TypeScript のため）
- 難しい場合は **SQL** でもよい。SQL の場合は**まとめてコミットできる形**にする（authored データであり PII を含まないため、コミットに支障がない）
- 選んだ言語は `metadata.json` の `tool.language`（`typescript | sql`）に記録する

## 構造（削除 → 投入 → 検証）

冪等性を担保するため、毎回この順で走る 1 つのエントリから実行する。

1. **削除**: **外部キーの依存の逆順**で対象データを削除する（子テーブルから先に消す）。事前削除により、何度実行しても同じ状態から始まる
2. **投入**: **外部キーの依存順**で投入する（親テーブルから先に入れる）
3. **検証**: 投入直後に FK 整合・必須項目・件数を自己検査する（詳細な検証観点は各フェーズのフロー）

## 冪等性の確認

- **2 回連続で実行し、同一状態になることを確認する**（件数・値が一致する）
- 追記型（既存を消さずに足す）にしない。再実行で件数が増えるものは冪等でない

## 決定論

- **ID・連番・UUID を固定値にする**（自動採番・ランダム生成をそのまま使わない）
- 時刻に依存するデータは `metadata.json` の `base_time` からの**相対時刻**で生成する（実行時刻を参照しない）
- これが無いと投入のたびに現行と新側で違う値になり比較が成立せず、再投入のたびにパリティスイートが壊れる

## 接続とシークレット

- 接続情報は設定の `current.db.env_vars` / `new.db.env_vars` の環境変数**名**経由で読む。**値を設定・コード・ログ・成果物に書かない**
- シークレットが要るコマンドは `secrets.wrapper` を前置する
- 投入前に本番でないことの確認ゲート（SKILL.md「厳守の制約」1）を通す
- `forbidden_actions` を尊重する

## フェーズ B のターゲット追加

フェーズ B では、同じ論理データを**新側スキーマへ写像**して投入するターゲットを追加する（論理データ自体は変えない）。写像の設計は [`phase-b.md`](phase-b.md) に従う。ツールの削除・投入・検証の構造は共通のまま、接続先と写像層だけが新側向けになる。
