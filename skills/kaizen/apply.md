# 学び適用ガイド

## フロー概要

`.kaizen/` 内の `status: pending` ファイルを根本原因・種別でグルーピングし、グループ単位で 1 つの対策を作成する。`multiagent-setup` スキルを活用して成果物を作成する。

## 手順

### 1. 学びファイルの一覧とグルーピング

```bash
ls .kaizen/*.md 2>/dev/null
```

`status: pending` のファイルがなければ「適用待ちの学びはありません」とユーザーに伝えて終了する。

ファイルを Read して内容を把握し、以下の観点でグルーピングする:

- 同じ根本原因を持つファイルをまとめる
- `type` が同じで対象ファイルや対策が共通するものをまとめる
- 明らかに無関係なものは別グループとする

グループ一覧をユーザーに提示し、グルーピングが適切か確認する。

### 2. グループごとに適用（繰り返し）

各グループについて:

1. グループ内のファイル一覧と統合した事象・根本原因・提案をユーザーに提示する
2. 適用するか確認する（承認 / 却下 / スキップ）
3. **承認された場合**: Step 3 へ進む
4. **却下された場合**: グループ内の全ファイルを `status: rejected` に更新して次へ
5. **スキップの場合**: そのまま次へ

### 3. 成果物の作成

`type` フィールドに応じて `multiagent-setup` スキルを使用して成果物を 1 つ作成する。

| type | 適用先 | multiagent-setup コンポーネント |
|------|--------|-------------------------------|
| `rule` | `.agents/rules/<name>.md` | rule |
| `skill` | `.agents/skills/<name>/` | skill |
| `hook` | `.claude/settings.json` 等 | hooks |
| `doc` | `AGENTS.md` / `CLAUDE.md` 等 | docs |
| `other` | ユーザーと相談して決める | — |

グループ内に複数ファイルがある場合、それら全ての内容を統合して 1 つの成果物に反映する。

`type: rule` の場合: 各ファイルの「提案」セクションを統合してルール文面を作成する。

### 4. 成果物の確認と後処理

1. 作成した成果物をユーザーに確認する
2. 問題なければグループ内の全ファイルの `status` を `applied` に更新する
3. 修正が必要な場合は修正後に更新する

### 5. 全グループ処理後の対応

すべてのグループを処理した後、適用済みグループについてユーザーに確認する:

- **このブランチで作業継続**: コミットして次の作業へ進む
- **Issue を作成**: グループ（= 対策）ごとに 1 Issue を作成し、別ブランチで詳細を詰める

## クリーンアップ

`.kaizen/` のファイルが増えすぎた場合、ステータスを指定して削除する。

### 削除オプション

| オプション | 削除対象 |
|-----------|---------|
| `--applied` | `status: applied` のファイルのみ |
| `--rejected` | `status: rejected` のファイルのみ |
| `--applied-and-rejected` | `status: applied` と `status: rejected` の両方 |
| `--all` | 全ファイル |

### 削除手順

1. 削除対象ファイルの一覧をユーザーに提示する
2. 確認を得てから削除する

```bash
# 例: applied なファイルを一覧して削除
grep -l "^status: applied" .kaizen/*.md 2>/dev/null
```

## `.kaizen/` の Git 管理

`.kaizen/` ディレクトリはコミット対象とする（`.gitignore` に追加しない）。

- `status: applied` / `status: rejected` の履歴が残り、改善の経緯を追跡できる
- チーム内で学びを共有・レビューできる
- `pending` のファイルはどのエージェントも参照できる（Claude Code / Codex / Copilot 間の学び共有）
