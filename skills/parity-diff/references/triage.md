# LLM トリアージ（crop 対・1 件ずつ・分類 3 値）

正規化（[`normalize.md`](normalize.md)）を生き残った候補だけをモデルに渡す。**モデルの仕事は分類だけ**——「差分があるか」は決定論的ツールが済ませている。

## 入力の作り方

- **1 件ずつ**渡す（まとめて渡すとモデルが探し始め、検出させることになる）
- 渡すもの:
  - 名前無し要素（画素経路のみで拾った候補）: **差分領域の crop 対だけ**（現側 crop・新側 crop）。全画面を渡さない
  - 名前付き要素が絡む候補: crop 対に加えて特性の差分テキスト（プロパティ値・状態・相対位置）を添える
- **crop の位置づけ**: crop は分類を助ける提示方法であって、検出させるための工夫ではない。検出は決定論的ツールが済ませている前提を崩さない。目的はモデルに探させないことと、全画面を縮小して渡すことによる解像度劣化を避けること

## 質問形式

- 聞くのは「**この差分は重要か**」だけ
- 分類は 3 値: **要対応 / 許容 / 環境ノイズ**（`normalize.md` の機械分類で未確定のまま残った候補＝未説明のものが対象）
- ユーザーから「スクリーンショットを見比べて違いを見つけて」と頼まれた場合も**目視検出を代替提供しない**。検出は画素経路（記録済み差分ツール＋ `pixel-crops.mjs` の crop 対）に任せ、モデルは crop 対の分類だけを担う、という分担を説明して誘導する

## 「許容」の確定

- **「許容」の確定にはユーザー承認が要る**（AskUserQuestion）。承認 UI には crop・特性差分など**判断材料を同梱する**（直前のテキストが見えている前提を置かない）
- 承認されたら記録先へ**非破壊追記**する:
  - クラス/トークン単位で表せる系統差 → `component_diffs`（宣言者は本来 `parity-replace`。ここで確定した系統差は `parity-replace` 側の宣言として整理する）
  - T が引けないインスタンス単位 → `component_diff_exceptions`（スキーマは [`normalize.md`](normalize.md)）
  - 環境揮発など意図的差異 → `intentional_diffs`
- **未承認なら未説明のまま残す**（`diff.md` の未説明差分。未収束として扱う）

## 環境ノイズの分類

- 「環境ノイズ」に分類する根拠は**ノイズ基準値との対比を記録する**（`metadata.json.noise_baseline` の該当 page/state/viewport の差分量と同程度であること）
- ノイズ基準値が無い組はノイズと断定せず、要確認として残す

## 出力

各候補の分類（要対応 / 許容 / 環境ノイズ / 未説明）と根拠を [`convergence.md`](convergence.md) の収束判定・`diff.md` の差分一覧へ渡す。**修正はここで行わない。**
